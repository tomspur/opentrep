Detailed Description

template<typename CharT, typename StorageT, typename SizeT>
struct containers::tst_detail::node_base< CharT, StorageT, SizeT >

Behaviourally, there are 2 1/2 types of nodes:
  • basic node stores a single character, and points on to paths for higher, lower and equal strings.

  • end nodes store a mapped type item and points to higher children -
    No lower or equal children exist, since end-of-string char is -inf.
  • In practice, erased nodes are valueless endnodes: they have only hikid, and are unlinked from tree at any opportunity.

We only ever return end-nodes to client code, and always wrapped in proxy types with desired behaviour (assignable and/or dereferenceable).

Invariants:

  • eqkid (nid + 1) is valid except at endnode (terminal or erased).
  • lokid is always non-link at endnode.
  • hikid is a valid link at reachable erased nodes. (reachable: follows or is linked from a reachable node) Storage-independent base: clients do all access/transactions on value member.
